# Platform API Docs

## 模块定位

这里是 `platform-api` 模块自己的文档入口。

如果你已经确定当前就是在推进 `platform-api`，优先从这里进入，而不是直接在 `docs/` 根层里找散落文档。

## 推荐阅读顺序

1. [模块总索引](index.md)
2. [Testing Workflow](guides/testing-workflow.md)
3. [API 设计与调用链](guides/api-design-and-flow.md)
4. `steps/` 下当前正在推进的 step
5. `testing-training/` 和 `testing-automation/` 下对应 step 的测试文档

## 当前目录怎么理解

- `index.md`
  - 模块总索引，负责进度、入口和历史承接
- `guides/`
  - 稳定专题知识
- `steps/`
  - 单步实现记录
- `testing-training/`
  - 测试设计训练材料
- `testing-automation/`
  - 测试自动化交付记录
